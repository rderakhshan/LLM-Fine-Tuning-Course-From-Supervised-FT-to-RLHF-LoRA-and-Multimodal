Big Picture — How Pre-training Works

◆ The Pipeline — End to End

Pre-training follows a linear data-to-weights pipeline executed at enormous scale across thousands of GPUs for weeks or months:

Collect raw text → Clean & filter → Tokenize → Chunk to context window → Autoregressive training → Base model checkpoint
  • 1 Collect: Crawl the internet (Common Crawl), ingest Wikipedia, books, research papers, code repositories, forums. Scale: tens of terabytes to petabytes of text.
  • 2 Clean & filter: Deduplicate (MinHash), remove low-quality pages (perplexity filtering), strip HTML, filter toxic/spam content, language-identify to keep target languages.
  • 3 Tokenize: Convert text to token IDs using a learned tokenizer (BPE, SentencePiece, or similar). Vocabulary size is typically 32K–128K tokens.
  • 4 Chunk: Split the tokenized corpus into contiguous blocks matching the model's context window (e.g., 2048, 4096, or 8192 tokens). Documents are concatenated with a special separator token; chunks that cross document boundaries are allowed.
  • 5 Train: Autoregressive language modeling — for each position, predict the next token given all previous tokens. Loss is cross-entropy over the vocabulary. Gradients are computed and backpropagated through all transformer layers.
  • 6 Checkpoint: Save model weights periodically. The final checkpoint is the base model, capable of continuing text but not following instructions.

■ Data Types & Sources

The pre-training corpus is a mixture of diverse text sources, carefully balanced to produce a general-purpose model:

  • W Web pages — Common Crawl (monthly snapshots of the internet), filtered and deduplicated. This is the largest source (60–80% of tokens for most models).
  • B Books — Project Gutenberg, copyrighted books (where licensed), self-published works. Provides long-form narrative and deep reasoning.
  • A Academic papers — arXiv, PubMed Central, Semantic Scholar. Scientific rigor and formal language.
  • C Code — GitHub public repositories (Python, JavaScript, C++, etc.). Teaches logical structure, syntax, and reasoning.
  • E Encyclopedic — Wikipedia dumps in multiple languages. Factual knowledge and structured exposition.
  • F Forums & Q&A — Reddit, Stack Overflow, Quora. Conversational and instructional language patterns.
  • M Multilingual — Non-English text to enable cross-lingual transfer. Most models use 30–100+ languages.

Data ratios matter — too much web text makes the model chatty and prone to hallucination; too much code makes it overly literal. Each research team has proprietary mixing ratios.

Common Crawl Wikipedia GitHub arXiv Reddit Multilingual

■ What the Process Actually Does

At the mathematical level, pre-training solves one objective: maximize the log-probability of each token given its preceding context across the entire corpus.

  • → Loss function: L = -∑ log P(tokeni | token1, ..., tokeni-1) averaged over all positions
  • → The transformer architecture processes the entire sequence in parallel (masked self-attention), making training efficient on GPUs/TPUs
  • → Training is distributed across hundreds/thousands of accelerators using data parallelism, tensor parallelism, and pipeline parallelism
  • → Optimizer: AdamW with cosine learning rate schedule, warmup, and weight decay. Batch sizes can be millions of tokens.
  • → Training continues for multiple epochs over subsets (the entire corpus may be seen 1–3 times), with the loss steadily decreasing
  • → Emergent abilities: as the model scales past certain thresholds (e.g., >6B parameters), in-context learning, chain-of-thought, and world modeling emerge without explicit training
Autoregressive Distributed training AdamW Cosine schedule Emergent abilities

■ What the Model Learns

The model's weights encode a compressed representation of the training distribution. Through the next-token prediction objective, it implicitly learns:

  • K World knowledge — facts, entities, dates, events, scientific concepts. This is what RAG systems try to supplement.
  • L Language competence — morphology, syntax, semantics, discourse structure, and style. It can produce fluent, grammatical text in dozens of languages.
  • R Reasoning patterns — logical chaining, analogies, step-by-step reasoning, and common-sense inference learned implicitly from the data.
  • C Cultural & social norms — conventions, biases, and perspectives present in the training text (both desirable and problematic).
  • T Transferable representations — the hidden states serve as rich feature extractors that can be adapted to downstream tasks with minimal fine-tuning data.

Critically, the base model cannot: follow instructions, hold conversations, refuse harmful requests, cite sources, or produce structured outputs. These require the later SFT and alignment stages.

Key Takeaways — Unsupervised Pre-training

Why It Matters CORE INSIGHT

Unsupervised pre-training is the foundational stage that produces the base model through self-supervised learning — the model labels its own data by predicting the next token. This stage is where the LLM earns its name: doing language modeling at large scale (trillions of tokens) is what makes it a Large Language Model.

Self-Supervised Mechanism

  • No human labels needed — the next word in the sequence is the label
  • Input: "Metformin improves insulin" → Target: "sensitivity"
  • This is why it's called self-supervised: the data generates its own supervision signal
  • Same mechanism is reused in non-instructional fine-tuning (domain adaptation)
Self-supervised Next-token prediction No annotation

Scale & Infrastructure Barrier

  • Requires internet-scale data: Common Crawl, Wikipedia, books, code, research papers
  • Demands massive GPU clusters — hundreds of thousands of GPU-hours
  • Only large organizations can afford it: Meta, OpenAI, Google, DeepSeek, Mistral
  • This is why practitioners never train from scratch — they take existing base models and fine-tune

What the Model Actually Learns

  • General knowledge about the world (facts, concepts, relationships)
  • Language mechanics: grammar, syntax, sentence structure
  • Reasoning patterns implicit in the training text
  • But it does NOT learn to follow instructions, hold conversations, or produce structured answers — that requires the next stages

Context Window & Chunking Strategy

  • Each model has a max context window (e.g., GPT-3: 2K tokens, GPT-4: 128K tokens)
  • Pre-training data is split into chunks that fit within this window
  • Chunking can be done by: paragraph boundaries, semantic meaning, or fixed token length
  • Proper chunking strategy is critical for effective domain adaptation fine-tuning
Context window Chunk size Token limits

Practical Implications for Fine-tuning

  • You cannot do this stage yourself — use pre-trained base models from Hugging Face
  • Base model naming: llama-3.1-8B (base) vs llama-3.1-8B-instruct (fine-tuned)
  • Domain adaptation does re-use the same next-token prediction mechanism
  • Understanding pre-training helps debug fine-tuning failures (e.g., base model gaps)

Stage 2 — Supervised Fine-tuning (SFT)

Two Distinct Goals
Domain-adapt or teach instruction-following — two data strategies, one parameter-efficient toolkit.
Non-instructional
Adapt model to a specific domain's language, tone, and terminology from plain text (PDFs, documents).
help model learn the language, tone, and terminology specific to your domain
Instructional
Teach the model to follow instructions, answer questions, and produce structured human-like responses.
to train the model to understand instruction and generate well-structured and relevant answer

⚖ Parameter Level HOW

Full fine-tuning updates all weights — requires multi-GPU. Partial fine-tuning trains a subset. Modern LLMs use PEFT — train only a small set of extra parameters while the base model stays frozen.

  • L LoRA — Low-rank adapters (A×B matrix) added to attention projection layers. The most widely used PEFT method.
  • Q QLoRA — LoRA applied on a 4-bit or 8-bit quantized base model. Dramatically reduces memory (fine-tune 65B on a single 48GB GPU).
  • D DoRA — Weight-decomposed LoRA. Separates magnitude and direction for better learning fidelity.
  • A Adapter Layers — Small bottleneck modules (down-project → up-project) inserted into each transformer block.
  • B BitFit — Only the bias terms of the model are trained. Surprisingly effective for small-domain shifts.
  • 3 IA³ — Infused Adapter by Inhibiting and Amplifying Inner Activations. Learns element-wise scaling vectors on keys, values, and FFN.
  • P Prefix Tuning — Learns virtual token embeddings prepended to the keys/values in each attention layer.
  • S Prompt Tuning — Learns soft prompt embeddings prepended to the input tokens (only at the input layer, not per-layer).
LoRA is the default choice QLoRA for memory-constrained Single GPU feasible

☷ Data Level WHAT

Non-instructional (domain adaptation): Plain text from PDFs, documents, books. Trained via the same next-token prediction mechanism as pre-training — the model learns domain-specific vocabulary, tone, and terminology.

  • → Example: Pharma PDF about Metformin → model becomes a pharma domain expert
  • → Data needs: chunking by paragraph/token limit, cleaning, formatting as text column
  • → Output: produces continuous text in-domain but does NOT learn to follow instructions

Instructional (chat/conversational): Question-answer pairs — the model learns to map a user instruction to a structured, helpful response.

  • → Formats: Alpaca (instruction + input + response), context/response, user/assistant, system/user/assistant
  • → Datasets: Alpaca, Open-Orca, Mental Health Counseling
  • → Data creation: manual annotation, expert labeling, or LLM synthetic generation (feed plain text to GPT and ask it to generate Q&A pairs)
  • → This is the stage that gives SFT its name — because the data has a human-curated label (the answer)

Stage 3 — Preference Alignment

Goal
Align model outputs with human expectations — make responses safe, helpful, polite, and honest rather than just factually correct.
to make the model polite, safe, helpful, and aligned with the human value — improve the safety and ethics of the model

⚙ RLHF

Reinforcement Learning from Human Feedback. A multi-step pipeline:

  • 1 Collect human rankings of multiple model outputs for the same prompt
  • 2 Train a reward model that predicts which output humans prefer
  • 3 Use PPO (Proximal Policy Optimization) to update the LLM to maximize the reward model's score while staying close to the original SFT model (KL penalty)

Pioneered by OpenAI's InstructGPT paper. Complex to train — requires 4 models (policy, reference, reward, value) in memory simultaneously.

PPO algorithm Reward model Human annotation InstructGPT

⚙ RLAIF

Reinforcement Learning from AI Feedback. Same RL framework as RLHF, but the preference labels are generated by an AI judge (e.g., a larger LLM) instead of human annotators.

  • → Much more scalable — AI can label millions of preferences at low cost
  • → Introduced by Anthropic in their "Constitutional AI" research
  • → Still uses PPO/reinforcement learning — only the annotation source changes
  • → Trade-off: may inherit the AI judge's own biases
Scalable AI-as-judge Constitutional AI Anthropic

⚙ DPO

Direct Preference Optimization. The current state-of-the-art. A supervised approach — no reward model, no reinforcement learning loop, just a clever loss function.

  • → Loss: σ(β · log(πθ(y+|x) / πref(y+|x)) - log(πθ(y-|x) / πref(y-|x)))
  • → Intuition: maximize the gap between how much the model prefers the chosen response vs the rejected one, relative to the reference (SFT) model
  • → β (beta) controls how strongly the model must favor chosen over rejected
  • → Much simpler to train — no reward model, no PPO instability

Frameworks & Tools

ℹ Hugging Face

The standard ecosystem. Every fine-tuning pipeline builds on these libraries:

  • T Transformers — load any model with AutoModelForCausalLM.from_pretrained()
  • D Datasets — load, split, map, and format training data
  • P PEFT — LoRA config, get_peft_model(), PeftModel.from_pretrained()
  • T TRL — SFTTrainer, DPOTrainer, preference tuning classes
  • B BitsAndBytes — 4-bit and 8-bit quantization for memory-efficient loading
  • A Accelerate — multi-GPU, FSDP, DeepSpeed orchestration

ℹ LLaMA Factory

All-in-one open-source fine-tuning project. Covers the full pipeline with minimal code.

  • → Supports 100+ model architectures (Llama, Mistral, Qwen, etc.)
  • → Provides both Web UI (drag-and-drop) and CLI (YAML config)
  • → Built-in support for SFT, RLHF, DPO, and more
  • → Handles dataset formatting, tokenization, and training loop
Web UI CLI Multi-framework

ℹ Unsloth & Axolotl

Unsloth: High-performance LoRA/QLoRA. Claims 2x faster training and 50% less memory usage through optimized CUDA kernels. Works as a drop-in replacement for HF trainers.

Axolotl: Streamlined YAML-driven fine-tuning framework. Supports full, LoRA, QLoRA, FSDP, and DeepSpeed. Popular for reproducible training configs.

Special Topics

SLM Fine-tuning

Fine-tuning Small Language Models (sub-3B parameters like TinyLlama, Phi-3, Gemma-2B) for resource-constrained or on-device deployment. Same techniques — LoRA, DPO, instruction tuning — applied at smaller scale.

Key challenge: small models have limited capacity, so data quality matters even more.

Multimodal Fine-tuning

Extending fine-tuning to vision-language models (LLaVA, Qwen-VL, InternVL). Involves aligning a vision encoder (e.g., CLIP, SigLIP) with a language model via a projection layer, then tuning on image-text instruction data.

Key difference: the loss is still next-token prediction, but the input now includes visual tokens.

Embedding Fine-tuning

Tuning embedding models for improved retrieval quality in RAG pipelines. Uses contrastive learning on paired (query, relevant document) vs (query, irrelevant document) data to pull relevant embeddings closer and push irrelevant ones apart.

Different from LLM fine-tuning — no next-token prediction, only representation learning.

Dataset Types

☶ Non-Instructional

Single plain text column. Each row is a chunk of text (paragraph, fixed token length). Used for domain adaptation and continued pre-training via next-token prediction.

  • → FineWeb — 15-trillion token web scrape (Hugging Face)
  • → Pile PubMed — biomedical research paper abstracts
  • → Custom PDFs — extract text, chunk by paragraph or token limit

☶ Instructional

Input + output pairs. Multiple column formats are supported: instruction/input/response, context/response, user/assistant, or system/user/assistant.

  • → Alpaca — 52K GPT-generated instruction-following examples
  • → Open-Orca — 4M+ examples with system prompts, questions, and responses
  • → Mental Health Counseling — context/response format for domain-specific chatbot training

☶ Preference / DPO

Prompt + chosen + rejected columns. The chosen response is the human-preferred one; the rejected is the dispreferred one. Used for DPO, RLHF, and RLAIF training.

  • → Anthropic HH-RLHF — 170K helpful/harmless dialogues with chosen/rejected pairs
  • → UltraFeedback — 64K prompts with 4 LLM responses each, rated across multiple criteria
  • → Math-Step-DPO-10K — step-level preference pairs for math reasoning
